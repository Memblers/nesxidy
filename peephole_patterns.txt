Peephole optimization patterns extracted from opt65.c

Phase 0 - simple dead code removal
- Delete any instruction with feeds==0 and not marked fixed (no side effects).

Phase 1 - constant tracking & trivial folding (opti1)
- Redundant loads/stores:
  * LDA #imm delete if A already imm and flags not used
  * LDX #imm delete if X already imm
  * LDY #imm delete if Y already imm
  * STA addr delete if memory value == A
  * STX addr delete if memory value == X
  * STY addr delete if memory value == Y
- Redundant flag arithmetic:
  * AND/SBC... #0, ORA #$FF, EOR #0 drop if flags not consumed
  * same tests against known register value
- Constant folding:
  * AND/ORA/EOR #imm with known A -> replace with LDA #result
  * same operations with memory operand when memory value known -> immediate or LDA
  * LD? addr where addr known constant -> convert to immediate load
  * CMP/CPX/CPY, ADC/SBC with known memory -> convert to immediate
  * AND/ORA/EOR with known memory -> convert to immediate or LDA
- Register-to-memory propagation:
  * STA addr followed later by LDA addr either deletes LDA or transforms STA/ LDA to TAY/TYA or TAX/TXA when possible
- Miscellaneous:
  * tax/txa pairs reduced to a single transfer
  * immediate conversion of loads from resolved addresses
  * track INC/DEX/INX/DEY etc to update reg values

Phase 2 - block-movement & register-usage rewrites (opti2)
- Move isinit instructions (mostly LDY and flag operations) upward past changable instructions
- STA ... / LDA ... pairs:
  * If Y unused between, replace with TAY/TYA
  * If X unused, replace with TAX/TXA
  * try to move blocks around using try_sim2 for more advanced cases
- TAX/TXA pairs: delete second TXA; if followed by STX and X unused, change STX->STA
- Move LDY #imm downward to enable other patterns

Phase 3 - adjacent-pair rewrites (opti3)
- Duplicate elimination: identical instruction/parameter sequences removed (unless flagged dupl==0)
- Specific pair rules:
  * TAX,TXA -> keep TAX
  * TAX,STX (X unused) -> STA
  * TXA,TAX -> keep TXA
  * TXA,STA (no A passes) -> STX
  * TAY,TYA -> keep TAY
  * TAY,STY (Y unused) -> STA
  * TYA,TAY -> keep TYA
  * TYA,STA (no A passes) -> STY
  * TXS,TSX -> keep TXS
  * TSX,TXS -> keep TSX
  * LDA,TAX (A not used/modified) -> LDX
  * LDA,TAY -> LDY
  * LDA,STA same addr -> delete STA
  * LDX,TXA (X unused) -> LDA
  * LDX,STX same addr -> delete second
  * LDY,TYA -> LDA
  * LDY,STY same addr -> delete second
  * STA,LDA same addr -> delete LDA
  * STA,LDX -> append TAX
  * STA,LDY -> append TAY
  * STX,LDX same addr -> delete second
  * STX,LDA -> append TXA
  * STY,LDY same addr -> delete second
  * STY,LDA -> append TYA

Additional helpers and notes:
- resolve_abs(), parsing numeric expressions
- add_dep(), make_feedlist() compute dependency bitmasks for regs/flags/memory
- try_sim2() performs block swapping for STA/ LDA patterns

Usage suggestion:
- maintain dependency map of A/X/Y/S/flags and memory addresses
- apply the above rules in order during a binary peephole pass
- cycle until no changes; track metrics like removed instructions


Unintended/opcode patterns (concrete sequences)
----------------------------------------------

Below are explicit rules you can implement in a binary peephole pass to either
*emit* or *recognise* illegal NMOS 6502 opcodes.  Each entry shows the legal
sequence and the corresponding fused opcode.

- `LDA addr` + `TAX` → **LAX addr** (opcodes A7/AF/B7/BF/A3/B3 depending on
  addressing).  Save one byte and one cycle.
- `LDX addr` + `TAX` or `LDA addr` + `TAX` (when X contains the same value) →
  **LAX addr** as above.
- `LDA addr` + `STX addr` (with A==X) → **SAX addr**.  (variants: 87/97/8F).
- `LDX #imm` + `STX addr` → **STX addr** already one byte; consider SAX for
  variants where A holds the same value.
- `LDA addr` + `ORA addr` → **SLO addr** (A = (M<<1) OR A, M updated).  Useful
  when you need both a shift and an OR on the same location.
- `LDA addr` + `ROL addr` → **RLA addr** (rotate‑and‑OR).
- `LDA addr` + `EOR addr` → **SRE addr** (shift‑and‑EOR).
- `LDA addr` + `ROR addr` → **RRA addr** (rotate‑and‑EOR/ADC behaviour).
- `LDA addr` + `LSR addr` → **ALR addr** (AND then LSR) or **ARR addr** (AND
  then ROR) depending on flags required.
- `LDA #imm` + `AND #imm` + `CLC` → **ANC #imm** (carry←A&imm, A←A&imm).
- any sequence `NOP`/`NOP` can be replaced by a multi‑byte DOP/TOP (04/44 etc)
  when you just need padding; the specific opcode corresponds to the length.

You can add similar entries for other undocumented opcodes such as XAA, AHX,
TAS, but the above cover the most frequently used fusions in demo‑scene code.

To integrate:
1. **Generation rule**: when emitting code, look for the legal pair and output
   the illegal opcode instead (watch addressing mode limitations).  Keep a
   flag to restrict this to NMOS‑only builds if you need portability.
2. **Recognition rule**: when analyzing existing ROM bytes, translate the
   illegal opcode to the equivalent legal sequence so the rest of your optimizer
   works correctly (and to spot places where you might have unknowingly
   generated one).

These patterns provide concrete cases you can add to `peephole_patterns.txt`
and implement in your own optimizer for extra byte‑saving on vintage hardware.
